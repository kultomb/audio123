# DIALOGUE STYLE RULES

## 1. CHARACTER VOICE PRESERVATION (CRITICAL)
- Each character has a unique way of speaking. Preserve their personality:
  - **Formal/authoritative character** → formal register in {lang}
  - **Casual/young character** → slang, contractions, informal particles
  - **Angry/emotional character** → sharp, short sentences; exclamations
  - **Sad/reflective character** → softer tone, longer pauses implied
- Match the emotional intensity of each line exactly.

## 2. STRICT TIMING & BREATH PACING
- **Film dialogue is timed to actor delivery.** Every block's duration must be respected.
- Short blocks (under 2 seconds) → extremely short phrases, often 1-3 words.
- Medium blocks (2-4 seconds) → one short sentence.
- Long blocks (4+ seconds) → one full sentence max.
- In dialogue, blocks often contain single words or short reactions ("Yeah.", "What?", "No way."). Translate these as equivalently short.

## 3. NATURAL CONVERSATION FLOW
- Use real spoken language: contractions, interruptions, overlapping-thought indicators.
- Translate interjections naturally: "Ừ", "À", "Ơ", "Này", "Thôi", "Thật á?".
- Pauses and hesitations: "ừm...", "ờ..." (only if in original).
- Question tags: "đúng không?", "phải không?", "nhỉ?", "hả?".

## 4. SELF-CONTAINED BLOCKS - BUT CONVERSATION-AWARE
- Each block is voiced separately. Make each one speakable alone.
- However, the conversation should flow naturally across blocks — like real back-and-forth.
- **NO ELLIPSIS BRIDGING**: Do NOT use `...` to connect blocks. End with natural punctuation.

## 5. FORMATTING
- Strict 1-to-1 block mapping. Never merge dialogue lines.
- Output ONLY valid SRT inside `<TRANSLATE_TEXT>` tags.

## 6. CRITICAL: COMPLETE ALL BLOCKS - NO DROPPING
- You MUST translate EVERY SINGLE block from the input. Do NOT skip, merge, or drop any blocks.
- The output must have EXACTLY the same number of blocks as the input.

## 7. CRITICAL: TIMESTAMPS - DO NOT MODIFY

- DO NOT merge blocks (which changes timestamps). Each block stands alone.

- Correct: `00:00:12,368 --> 00:00:16,704` (identical to input)
